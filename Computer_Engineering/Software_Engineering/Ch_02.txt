1. 소프트웨어 프로세스 모델

폭포수, 프로토타입, 나선형 모델, 애자일 프로세스, RUP(중요)

01. 폭포수 모델 (중요)
장점
 - 선형순차적 모델(중요)
 - 전 단계를 끝내야 다음 단계가 진행됨, 단계별로 정형화된 접근 방식(중요)
 - 체계적인 문서화 및 산출물 체크를 통한 프로젝트 진행의 명확화
 - 요구사항이 잘 이해되고 시스템을 개발하는 동안 급격하게 변경되지 않는 경우에 사용
 - 기술적 위협이 없고, 유사한 프로젝트 경험이 많아 비교적 정확한 비용 예측과 일정계획 하에 프로젝트를 추진할때 적합
단점
 - 요구사항 이해가 필요
 - 작업 지연에 따른 전체 일정 지연
 - 문서화에 대한 부담 가중
 - 초기 단계에서의 결저은 고객의 요구사항의 변경이 어려움

02. 프로토타입 모델 (중요)(컨셉_
- 사용자의 요구사항을 정확하게 파악하기 위해 실제 개발될 소프트웨어에 대한 견본(시제)품을 만들어 최종 결과물을 예측하는 모델
- 요구사항 수집 -> 빠른 설계 -> 프로토타입 구축 -> 고객 평가 -> 프로타입 구정 -> 구현
- 구현이 끝난 후 요구사항 수집 후 반복

장점
 - 사용자 요구사항 빠른 수용
 - 최종결과물의 일부를 볼 수 있음
단점
 - 시제품에 대한 요구사항의 증가
 - 부족한 문서화에 따른 유지보수의 어려움 

05. 나선형 모델 (중요)
- Bohem이 제안
- 폭포수 모델 + 프로토타입 모델 + 위험분석 기능(리스크 매니지먼트)
- 니선을 따라 돌듯 여러번의 소프트웨어 개발 과정을 거쳐 점지적으로 개발

장점
 - 가장 현실적인 모델
 - 위험분석 기능을 추가해 퀄리티 높은 소프트웨어 개발 가능
단점
 - 위험을 다룰 수 있는 전문가 필요

2. 애자일 소프트웨어 개발(Agile Software Development)

철학 - 문서보다 개발에 집중하자.
- 의사소통과 상호작용
- 문서화보다 개발에 집중
- 이해당사자와의 협업/협력
- 변경된 일정에 빠른 적응

문제기술서(Problem Statement)
 - 사용자가 경험할 수 있는 일종의 시나리오인 "사용자스토리" 개념
 - "스토리 카드" 작성
 - 관련되는 업무 영역끼리 분류

우선순위에 따른 개발 정리 : 개발에 대한 업무 선택 / 문제기술서 선택

애자일 프로젝트 관리
- 스크럼 용어
제품 백로그 - "해야 할 일(to do)" 목록
스프린트 - 반복
제품 소유권자 - 우선순위 정함

Scrum sprint cycle
 - 백로그에서 우선순위 항목 선택
 - 업무에 우선순위를 위해 미팅
 - Sprint(반복) 작업을 하여라
 - 리뷰 후에 반복
